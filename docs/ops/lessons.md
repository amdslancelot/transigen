# 踩雷教訓（追加不改舊條目，格式見 maintenance.md）

## 2026-07-06 交辦「追加檔案內容」時，分隔標記被 builder 貼進檔案

- 症狀：worklog.md 出現三處「（追加內容到此結束）」提示詞殘留；且其中一次追加沒有貼在檔案尾端，插進了中段。
- 錯誤路徑：交辦 prompt 用「要追加的內容（從下一行開始到結束）：…（追加內容到此結束）」描述邊界，haiku builder 把結尾標記當成內容一起貼上，並自行決定插入位置。
- 正解：主對話讀檔確認後直接用 Edit 修正（移除殘留、搬移段落）。
- 規則化：交辦「貼內容進檔案」的任務時，內容邊界用 markdown code fence 包住並明說「fence 本身不是內容」；驗收條件加一條「grep <標記文字> 在目標檔為 0 hits」與「新內容起始行號 > 原檔末行號」。追加類任務的回報若含矛盾行號，主對話必須自己讀檔驗證再回報使用者。
